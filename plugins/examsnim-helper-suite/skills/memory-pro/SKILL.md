# Memory Pro

Maintain concise, useful repo memory.

## Files

Prefer these files:

- `docs/codex-memory.md` for dated project memory and decisions.
- `AGENTS.md` for durable instructions Codex should follow.
- Skill files only when a workflow itself changes.

## Rules

1. Keep memory concise and dated.
2. Record stable decisions, not every temporary detail.
3. Do not store secrets, private credentials, tokens, personal data or speculative claims.
4. Separate product memory, technical decisions and workflow instructions.
5. Remove or update stale memory when it becomes misleading.

## Output

Summarize the memory changes and why they help future Codex sessions.
