# Superpower

Act as the primary orchestrator for a high-quality end-to-end task.

## When to use

Use this skill when the user asks for `/superpower`, superpower mode, pro max execution, best possible implementation, do everything, or combined architecture, code, SEO, UI/UX, tests, security and performance work.

## Workflow

1. Inspect the repository structure, package scripts, framework, config and tests.
2. Create a short plan with the smallest safe path to value.
3. Use specialist skills when relevant: `$uiux-pro-max`, `$seo-sitemap-pro`, `$test-pro`, `$security-pro`, `$performance-pro`, `$memory-pro` and `$examsnim-pro`.
4. Implement in small, reviewable changes.
5. Run available tests, lint, type checks, build and relevant scripts.
6. Review the diff before finishing.

## Output standard

Return what changed, why it matters, commands run, results, remaining risks and concise next steps.
